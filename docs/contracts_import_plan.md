**✅ Этап A выполнен 17.07** (`scripts/import-contracts.mjs --commit`, идемпотентен): 20 контрагентов, 22 договора (истёкшие с `is_active=false`), 38 прайс-строк (effective-dated, допники отдельными строками), 23 цены ГСМ, привязано 38 машин (+8 новых из приложений: 967 ALF, 248 ALI, 748 GH, 796 ANL, 028 AJM, 143 ABE, 664 AKX, 339 AMW), 13 водителей Тұрнияза (5 сматчены с записями из «ГСМ Карточка» по транслитерации и переименованы в полное ФИО), дозагружены 2 рейса 664 AKX за 12.07. `798 BH`/`796 BH` из договора отождествлены с `798 GH`/`796 GH` из БД (марки совпадают). Оценка по действующим договорам: УГ ~13,7 млн ₸ (601 рейс), Тұрнияз ~7,4 млн ₸ (433+2 рейса), Орынтай ~1,17 млн ₸ (67 ч), Харченко ~0,97 млн ₸ (80 ч), Горбань 0 (263 ANL в июльском табеле не встречался). Покрытие фактов: рейсы 1 034/2 009, смены 73/440 — остальное ждёт договоров этапа B. ⚠ Часы машин УГ до 08.07 (~часть из 444 ч) старше ставки нового договора — в settlement попадут как «нет тарифа», при необходимости их можно отнести к старому договору 27/03-11.

Исходный план этапа A:
1. `contractors`: ~18 контрагентов с ИИН/БИН (`bin`), руководителем (`head_name`), банком, `vat_payer` («с учётом НДС» → true, «без учёта» → false).
2. `contracts`: все договоры включая истёкшие (история для сверки периодов): number, type (`УОП→transportation`, `ПУСТ→equipment`), billing (`15days`/`monthly`), `valid_from` = дата договора, `valid_to` = «действует до».
3. `price_list` (effective-dated): базовая ставка с valid_from договора; допсоглашение = новая строка с датой допника (Yasina: 20 900→24 824→12 000; Бекнұр: 12 000→10 344,83; Алитурлиев: 15 700). Для ПУСТ — override `vehicle_id` (263 ANL→8 000, 412/413 AOD→17 500, 852 AOD→12 100).
4. `contract_fuel_prices`: 337/328/327/291/282 по договорам, effective-dated по допникам.
5. `vehicles`: проставить `contractor_id`+`contract_id` 38 машинам (действующие договоры приоритетнее истёкших), уточнить brand (SITRAK, SHAANXI, SCANIA), создать ~9 новых машин из действующих приложений.
6. `drivers`: 13 водителей Турнияза → contractor_id, дедуп с существующими (Abdibaev и т.п.); водители из прил. Услуги Грузов не заполнены в docx.
7. Дозагрузка факта: 2 рейса 664 AKX за 12.07 (лист «Рейсы» r127).
8. Сверка: `loadSettlement` по Услуги Грузов 08/07 и Орынтай — суммы должны построиться без «нет тарифа» по их машинам.

**Этап B — после ответов заказчика:** договоры на непокрытые машины, нормы расхода л/час, Алитурлиев (номер грейдера), судьба истёкших (пролонгации).

**Замечание по схеме:** ничего менять не нужно — все сущности ложатся в текущие таблицы. Истёкшие договоры получат `is_active=false`.
