Контекст: система QuarryOps уже развёрнута и работает (Дашборд, Табель, Рейсы, Выдача, Бензовоз, Журналы, Закрытие, Документы, Справочники). Архитектуру не меняем. Нужно добавить прогнозный блок поверх существующих данных.

ЗАДАЧА

1. Справочники → новая карточка «Параметры прогноза» (ввод один раз, редактируемо):
   - Объём на начало периода: дата + значение, м³ (пример: 01.07.2026 → 150 000 м³)
   - Целевой объём: значение, м³ (пример: 500 000 м³)
   - Целевая дата завершения: необязательное поле — если заполнено, включается сценарий 6.

2. Дашборд → вкладка «Работа», новый блок «Прогноз» (рядом с «Моточасы»), считается автоматически из уже существующих данных Рейсов + чисел из п.1:

   a) График накопительного объёма по дням: линия «Факт» (старт + сумма кубов из Рейсов нарастающим итогом по датам) и линия «План» (прямая от старта до цели/даты).

   b) Текущий темп производства: скользящее среднее за 7 / 14 / 30 дней (м³/сутки), плюс мин/медиана/макс за тот же период — не одно число, темп реально скачет от 0 (простой) до 10 830 м³/сутки.

   c) Прогноз даты завершения при сохранении темпа — 3 сценария: оптимистичный (по лучшему устойчивому темпу), базовый (по среднему), пессимистичный (по последним N дням с текущими простоями).

   d) Остаток: сколько м³ осталось до цели и сколько дней по каждому из трёх сценариев.

   e) % выполнения от целевого объёма.

3. Развилка «задана целевая дата» (если поле из п.1 заполнено):
   - Система считает требуемый темп = (Цель − Текущий объём) / (дней до целевой даты).
   - Сравнивает его с текущим фактическим темпом (п.2b) → показывает разрыв в м³/сутки.
   - Переводит разрыв в дополнительные самосвало-рейсы/сутки (по факту ~19 м³ на рейс — брать фактическое среднее по Рейсам, не константу).
   - Через соотношение экскаватор : самосвал = 1:10 (редактируемое поле в Справочниках, по факту подтверждается сменными нормами 3/30 и 5/50) считает, сколько дополнительной техники (самосвалов и экскаваторов) нужно вывести на линию сверх уже работающей, чтобы уложиться в срок.
   - Явно маркирует расчёт как оценочный диапазон (мин–max), а не точное число: полной картины по фактической доступности техники (ОЕЕ — простои, ремонты) пока нет. Коэффициент доступности по умолчанию ~0.75, редактируемый, в дальнейшем пересчитывается по факту простоев из Табеля/Журналов.
   - Результат — на арендуемую технику: «нужно дополнительно N–M самосвалов и K–L экскаваторов, чтобы завершить к [дата]».

4. Блок «Расстановка техники» (сейчас в системе отсутствует — новая вкладка/под-вкладка, например в Дашборде рядом с «Сегодня / Топливо / Работа»):
   - Схематичная карта объекта по зонам: карьер — технологическая дорога (плечо, редактируемая длина, по умолчанию 6.5 км) — полотно.
   - В каждой зоне — иконки/счётчики техники по типам, с количеством и средней производительностью за период (из Рейсов), т.е. не просто «где стоит», а «сколько сделала» по зоне.
   - Типовая логика зон (задаётся в Справочниках, редактируемо, не хардкод): экскаваторы — карьер; самосвалы — челнок карьер↔полотно; поливочные машины — полотно; бульдозеры — и карьер, и полотно; катки — полотно; грейдеры — дорога.
   - Сбоку — список прочей техники без географической привязки (счётчик, статус: работает/простой/ремонт — из Табеля).
   - Ключевые параметры производства на этом же экране: текущий темп (из п.2b), соотношение экскаватор:самосвал факт/норма, средний объём на рейс, доля ПРС от общего объёма.

5. Источник факта для расчёта темпа — только данные из вкладки «Рейсы» (не из Табеля — Табель даёт «кто вышел», а не «сколько перевезено»). Статус техники «в работе» на карте (п.4) и в блоке «по технике» — производное поле: есть в Табеле на смену И есть рейсы за этот период → «в работе»; есть в Табеле, но 0 рейсов → простой/аномалия; нет в Табеле → не выведена.

6. Дни с 0 рейсов не должны обнуляться молча — если день отмечен как простой (погода/техника), при расчёте скользящего среднего его нужно явно исключать из знаменателя «рабочих дней» либо показывать отдельным флагом на графике, иначе средний темп занижается искусственно.

7. Сверху блока «Прогноз» — сводка на 5 чисел для показа руководству без графиков:
   - Выполнено: X из 500 000 м³ (Y%)
   - Текущий темп: Z м³/сутки (за последние 14 дней)
   - Прогноз завершения при текущем темпе: дата–дата (диапазон трёх сценариев)
   - Если задана целевая дата: отставание/опережение ±N дней и требуемая доп. техника (из п.3)
   - Текущее соотношение экскаватор:самосвал (факт vs норма 1:10)

ИСТОРИЧЕСКИЕ ДАННЫЕ
Прилагается файл istoricheskie_dannye_m3.csv — очищенная и структурированная выгрузка фактических рейсов/объёмов за 21.05–15.07.2026 (дата, смена, поток грунт/ПРС, рейсов, м³, статус дня, примечания с флагами исправленных опечаток источника) для первоначальной загрузки в Рейсы, чтобы прогноз считался не с нуля, а на реальной истории.
