## 3. Качество данных и правила нормализации

### Госномера (главная боль)
- Разные написания: `013AKD` / `013 AKD` (пробелы), `305 AlZ` (строчные), `750 АЕВ` (кириллические омоглифы А/Е/В), `AHO 374` = `374 AHO` (порядок).
  **Правило:** upper → убрать пробелы → кириллица→латиница (АВЕКМНОРСТУХ) → канонический порядок «цифры+буквы».
- Excel-числа вместо номеров: `0.13`→`013 …`, `0.89`→`089 AMD`, `0.29`→`029 AOD`, `0.87`→`087 AKD` (ввели с запятой). Восстанавливать по коду техники + известному парку.
- Обрезанные: `252`, `695`, `413`, `769`, `380`, `370`, `552`, `795` — дополнять по коду техники (напр. `252`+K = `252 AKD`, но есть и `252 ALD`+E — неоднозначность!).
- Мусор: `VOLVO` (марка вместо номера), `AKD 595D`, `529M.C`, `GK`/`GH`/`664` — на ручной разбор (~20 строк).

### Часы (Журнал)
- `remont` ×1 → **не грузим** (решение заказчика: простои не заводим).
- `0` часов ×2 и пустые ×4 → пропустить.
- `8'45` ×1 → трактовать как 8.75 ч.
- Дата-строка ` 03.07.2026` ×1 → `2026-07-03`.
- Дубли (нарушат `unique(vehicle_id, shift_date, shift_type)`): 4 пары — `710 ABL` и `176 ADM` за 04.07, `595 AKD` и `089 AMD` за 12.07. Правило: суммировать? брать максимум? — уточнить (по 10+10 похоже на день+ночь → вторую писать как shift_type='night').

### ИП-владельцы
Колонка «ИП» заполнена только в блоке за 13.07 — даёт связку машина→владелец для 10 машин: Абитов (597 AND), Баймурзин (632 AND), Балгазин (595 AKD), Шугуров (795 ABD), Медыбаев (412 AOD), Узакбаев (413 AOD), Койшыбаев (178 AXD), Бисенов (769 AND), Нуспенов (252 AKD), Саликов (089 AMD). Для остальных ~130 машин владелец неизвестен → `contractor_id = null` до получения реестра.

### Водители (ГСМ Карточка)

## 4. Пробелы схемы (нужна миграция перед импортом)

1. **`shift_records.driver_id NOT NULL`** — в табеле водителей нет. → сделать nullable.
2. **`fuel_issues.driver_id NOT NULL` и `driver_signature_url NOT NULL`** — в исторических данных нет ни водителя (лист «Бензовоз»), ни подписей. → сделать nullable; целостность «живых» записей сохранить check-ом по `entry_source`.
3. **`trip_records` не имеет бизнес-даты** (только `created_at`) и требует `driver_id`/`route_id`. → добавить `trip_date date`, ослабить `driver_id`; суточный итог разворачивать в N строк (2 024 шт.).
4. **Идемпотентность импорта**: у `fuel_issues`/`trip_records` нет натурального ключа. → добавить во все 4 таблицы `import_ref text unique` (формат `july16:<лист>:<строка>[:n]`) + `entry_source text default 'app' check (entry_source in ('app','import'))`. Повторный запуск = `on conflict (import_ref) do nothing`.
5. `hours > 0` в shift_records — строки с 0 часов не грузим, конфликт снят.
